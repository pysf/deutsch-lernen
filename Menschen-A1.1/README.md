# Menschen A1.1 — Vocabulary Study Site

Offline German vocabulary pages for the **Menschen A1.1** (Hueber) textbook,
with listen-and-repeat MP3 audio generated via Google Cloud Text-to-Speech.

---

## Project structure

```
Menschen-A1.1/
├── index.html              # lesson index page
├── Lektion-1.html          # vocabulary pages (1–12)
│   …
├── Lektion-12.html
├── Lektion-10.mp3          # audio files (generated — one per lesson)
│   …
├── Lektion-10.ssml         # SSML source for the audio (generated by tts.py)
├── tts.py                  # audio generation script
├── Makefile                # sets up venv, installs deps, runs tts.py
├── requirements.txt        # google-cloud-texttospeech
├── prompt_html.md          # full spec for generating HTML pages
├── prompt_audio_google.md  # full spec for generating audio scripts
└── voice-samples/          # short voice comparison samples (Kore, Leda, etc.)
```

## HTML vocabulary pages

Each lesson page (`Lektion-N.html`) is a self-contained, offline-ready,
mobile-friendly vocabulary page. No external dependencies.

### Features
- **Color-coded word types**
  - Nouns: der = blue, die = red/pink, das = green
  - Verbs: regular = orange, irregular = purple
- **Dictionary-style POS markers** after each word: `(m)` `(f)` `(n)` `(t)` `(i)` `(adj)` `(adv)`
- **Perfekt form** shown for every verb: `einsteigen · ist eingestiegen`
- **Two sample sentences per entry** on one line, separated by ` · `
  - Verbs: present tense · Perfekt
  - Nouns: nominative · accusative (shows article change, e.g. der → den)
  - Adjectives / adverbs: single sentence
- MP3 download link at the top of each page

See **`prompt_html.md`** for the full spec and HTML patterns.

## Audio files

Each MP3 is a **listen-and-repeat** exercise:

1. Lesson title (male voice)
2. For each vocabulary item:
   - Word spoken **3 times** at 85% speed by a female voice (2.5s gap between reps; 3rd rep has falling "last one" intonation)
   - First sample sentence at normal speed by a male voice
   - 4s pause (time to repeat)

### Voices
| Role | Voice |
|------|-------|
| Word / drill | `de-DE-Chirp3-HD-Kore` (female, 15% slower) |
| Sentence / context | `de-DE-Chirp3-HD-Charon` (male, normal speed) |

Google Chirp3 HD is the most natural German voice tier and supports SSML
`<break>` tags. Free tier: 1M chars/month (~20× a full lesson set).

See **`prompt_audio_google.md`** for the full spec and SSML format.

## Generating audio

### One-time setup
```bash
# Install gcloud CLI (if not already installed)
brew install --cask google-cloud-sdk

# Authenticate and set your GCP project
gcloud auth application-default login
gcloud auth application-default set-quota-project YOUR_PROJECT_ID
gcloud services enable texttospeech.googleapis.com --project=YOUR_PROJECT_ID
```

### Render a lesson
Edit `tts.py` — set `LESSON`, `TITLE`, and `ITEMS` for the lesson you want.
Then run:

```bash
make
```

This creates `.venv`, installs `google-cloud-texttospeech`, and runs `tts.py`.
Output: `Lektion-N.ssml` (SSML source) and `Lektion-N.mp3` (final audio).

### What `tts.py` does
- Builds SSML from the structured `ITEMS` list
- Writes `Lektion-N.ssml` as a human-readable source file
- Auto-splits SSML into chunks ≤ 4500 bytes (Google's per-request limit)
- Calls the API per chunk, concatenates MP3 output

---

## Adding a new lesson

1. **HTML page** — follow `prompt_html.md`. Use the existing lessons as templates.
2. **Audio** — update `LESSON`, `TITLE`, and `ITEMS` in `tts.py`, then `make`.
3. **Index** — add the lesson to `index.html`'s `<ul class="lektion-list">`.
